## Обзор обучающей программы по Loguru (подробно и “по‑человечески”)

Эта документация написана так, чтобы в ней разобрался человек,
который **только начинает** знакомство с логированием и библиотекой **Loguru**.
Задача текста — объяснить не только “что делает код”, но и **зачем это сделано именно так**.

---

### 1. Что вообще делает программа `guide_loguru.py`

`guide_loguru.py` — это **учебная демонстрация** Loguru.
Вы запускаете её одной командой:

```bash
python guide_loguru.py
```

И получаете:

- красивый лог в консоли (с цветами, временем, уровнем, местом в коде и контекстом `extra`);
- несколько файлов в папке `logs/`:
  - текстовый лог с ротацией/хранением/сжатием;
  - JSON Lines (`.jsonl`) для машинной обработки;
  - отдельный файл с фильтрацией (например, только записи с `topic="db"`).

Главная цель программы — **показать на живом примере**, как устроена настройка Loguru:
sink’и, формат, уровни, контекст, исключения, фильтры и т.д.

---

### 2. Общая архитектура программы (из каких блоков она состоит)

Внутри `guide_loguru.py` есть несколько логических частей:

1. **Sink-функция `sink_print(message)`**
   Она принимает уже сформированное Loguru сообщение и печатает его в консоль.
   Этот приём позволяет контролировать вывод, при этом не импортируя другие модули.

2. **Функция `section(title)`**
   Печатает “разделитель” (рамку) и заголовок раздела, чтобы вывод был читаемым.

3. **Настройка вывода**
   - `configure_console()` — убирает дефолтные sink’и и добавляет консольный sink с красивым форматом.
   - `configure_files()` — добавляет несколько файловых sink’ов: текстовый лог, JSON‑лог, лог с фильтрацией.

4. **Набор демо-функций**
   Каждая демонстрация — отдельная функция:
   - `demo_basics()` — базовые уровни (`DEBUG/INFO/WARNING/ERROR/CRITICAL`);
   - `demo_custom_level()` — кастомный уровень;
   - `demo_formatting_and_lazy()` — форматирование и `opt(lazy=True)`;
   - `demo_bind_and_context()` — `bind()` и `contextualize()`;
   - `demo_filtering()` — фильтрация по `extra`;
   - `demo_exceptions()` — `logger.exception()` и `@logger.catch`;
   - `demo_patch()` — `patch()` для изменения записи;
   - `demo_disable_enable()` — `disable()/enable()` для модуля.

5. **Точка запуска `run()`**
   Последовательно конфигурирует Loguru и запускает все демонстрации.

---

### 3. `configure_console()` — зачем remove() и как устроен format

В Loguru обычно делают так:

- `logger.remove()` — убрать дефолтную конфигурацию (чтобы не получить дублирование);
- `logger.add(..., format=...)` — добавить свой sink и формат.

Формат в проекте специально сделан “информативным”: показывает время, уровень, место в коде и `extra`.
Это помогает понять, что лог — это не только “строка сообщения”, но и набор полей.

Опции `backtrace=True` и `diagnose=True` усиливают вывод ошибок: стек и детали становятся понятнее.

---

### 4. `configure_files()` — несколько файловых sink’ов

В проекте добавляются три примера:

1. **Текстовый файл** в `logs/`:
   - ротация по размеру (`rotation`);
   - хранение ограниченного количества файлов (`retention`);
   - сжатие архивов (`compression`).

2. **JSON Lines** (`serialize=True`):
   - удобно для дальнейшего анализа (например, парсить как JSON по строкам).

3. **Фильтрация**:
   - отдельный файл для сообщений с определённым контекстом `extra` (например, `topic="db"`).

---

### 5. Демо “lazy” — почему это важно

В `demo_formatting_and_lazy()` сравнивается:

- обычный вызов, где значение вычисляется **до** вызова логгера;
- `logger.opt(lazy=True)`, где callable вычислится **только если** запись реально будет залогирована.

Это важно для производительности: тяжёлые вычисления не должны происходить, если уровень логирования их всё равно “отрежет”.

---

### 6. Контекст и фильтрация — практическая идея

`bind()` и `contextualize()` добавляют `extra`‑поля к записи.
Это позволяет:

- помечать запросы (`request_id`), пользователей (`user_id`), компоненты (`topic`);
- фильтровать логи по этим полям и писать “поднабор” в отдельный файл.

---

### 7. Итог: чему конкретно учит эта программа

1. Как быстро настроить Loguru для консоли и файлов.
2. Как проектировать формат логов: что выводить и зачем.
3. Как добавлять контекст и использовать его для фильтрации.
4. Как правильно логировать исключения.
5. Как делать логи удобными и для человека, и для автоматической обработки (JSON).
