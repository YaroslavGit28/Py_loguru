# Техническое задание

## 1. Описание задачи

Нужно разработать учебный мини‑проект на Python, который демонстрирует работу библиотеки **Loguru** на реальном примере. Проект должен содержать исполняемый скрипт `guide_loguru.py`, который последовательно показывает основные возможности Loguru: уровни логирования, форматирование, контекст (`extra`), обработку исключений, фильтрацию, патчинг записей, отключение логов и запись в файлы (включая JSON) с ротацией/хранением/сжатием.

Проект должен быть оформлен так, чтобы по нему можно было **и запустить демонстрацию**, и **изучить подходы к логированию**.

## 2. Цель работы

Цель работы — получить практический опыт использования Loguru и сформировать инженерное понимание того, как строить удобные и информативные логи:

- быстро настраивать вывод (консоль и файлы);
- управлять уровнями и форматами;
- добавлять контекст к логам;
- корректно логировать исключения;
- делать логи пригодными для анализа человеком и машиной.

## 3. Функциональные требования

1. Проект написан на Python 3.x и использует библиотеку `loguru`.
2. Должен быть скрипт `guide_loguru.py`, который запускается командой:
   - `python guide_loguru.py`
3. Скрипт должен демонстрировать минимум следующие возможности:
   - стандартные уровни (`DEBUG/INFO/WARNING/ERROR/CRITICAL`);
   - создание кастомного уровня (`logger.level(...)` и `logger.log(...)`);
   - форматирование сообщений и плейсхолдеры Loguru;
   - “ленивые вычисления” через `logger.opt(lazy=True)` (наглядное сравнение);
   - контекст: `bind()` и `contextualize()`;
   - фильтрацию логов через `filter=...` (например, по `extra["topic"]`);
   - обработку исключений через `logger.exception()` и `@logger.catch`;
   - модификацию записи через `patch()`;
   - отключение/включение логов для модуля через `disable()/enable()`.
4. Скрипт должен писать логи:
   - в консоль (через sink‑функцию);
   - в файлы в папку `logs/` (текстовый лог с ротацией/retention/compression);
   - в JSON Lines (`.jsonl`) через `serialize=True`;
   - в отдельный файл с фильтрацией (например, `logs/db_only.log`).
5. Вывод (в консоль и/или файлы) должен быть читаемым: разделы, подписи, понятные сообщения.

## 4. Нефункциональные требования

1. **Понятность**: сообщения и разделы должны быть написаны так, чтобы их понимал начинающий.
2. **Читаемость кода**: демонстрации должны быть вынесены в отдельные функции и логически сгруппированы.
3. **Портируемость**: запуск на Windows / Linux / macOS при наличии Python 3.x.
4. **Минимум зависимостей**: достаточно одной зависимости `loguru` (через `requirements.txt`).
5. **Документированность**: проект содержит `README.md` и документацию в `docs/`.

## 5. Ограничения и допущения

1. По условию задачи, код в `guide_loguru.py` **не импортирует ничего, кроме `loguru`**.
2. Не требуется сложная бизнес‑логика: проект — демонстрационный.
3. Допускается, что папка `logs/` создаётся автоматически Loguru при первом запуске.
4. Демонстрация рассчитана на локальный запуск одним пользователем.
