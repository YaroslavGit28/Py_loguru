# LOGURU Guide (пособие-проект)

Этот мини‑проект — практическое пособие по **Loguru**: _зачем_, _как_ и _почему_ его используют вместо (или поверх) стандартного логирования.

## Зачем Loguru

- **Быстрый старт**: `from loguru import logger` — и можно логировать сразу, без долгой конфигурации.
- **Удобная конфигурация**: один метод `logger.add(...)` настраивает вывод в консоль, в файл, JSON‑логи, ротацию, хранение, компрессию и т.д.
- **Читаемые ошибки**: встроенные улучшения трейсбеков (`backtrace`, `diagnose`) и удобный `logger.exception()` / `@logger.catch`.
- **Структурирование**: `bind()` / `contextualize()` позволяют добавлять контекст (например, `request_id`, `user_id`) в каждый лог.

## Как запустить

1) Установите зависимости:

```bash
pip install -r requirements.txt
```

2) Запустите демонстрацию:

```bash
python guide_loguru.py
```

После запуска появится папка `logs/` с примерами:
- обычные текстовые логи с ротацией/удержанием/архивацией
- JSON Lines (`.jsonl`) для машинной обработки
- логи с фильтрацией по контексту (`extra`)

## Как устроен проект

- `guide_loguru.py` — **один исполняемый скрипт**, который импортирует **только** `loguru` и последовательно показывает основные возможности.
- Внутри скрипта все примеры сгруппированы по темам, а вывод снабжён пояснениями прямо в логах.

## На что смотреть в выводе

- **Форматирование**: плейсхолдеры вроде `{time}`, `{level}`, `{message}`, `{extra[...]}`.
- **Уровни**: стандартные (`DEBUG/INFO/...`) и кастомный уровень.
- **Контекст**: `bind()` и `contextualize()` добавляют `extra`‑поля.
- **Исключения**: `logger.exception()` и `@logger.catch` показывают стектрейс.
- **Файлы**: `rotation`, `retention`, `compression`, `serialize`.

## Ограничение по задаче

По требованию задачи, код в `guide_loguru.py` **не импортирует ничего, кроме `loguru`**.
